“前端性能优化”

发表于 2018-09-21 | 分类于 原生JavaScript

网页的加载

网页加载大致分为五步：

  1. HTML代码转化为DOM
  2. CSS代码转化为CSSDOM
  3. 结合DOM和CSSOM，生成一棵渲染树
  4. 生成布局（layout），即将所有渲染树的所有节点进行平面合成
  5. 将布局绘制（paint）在屏幕上

网页加载性能消耗主要在第四与第五步，这两步合称为“渲染”。

重排和重绘

在网页生成的时候，至少会渲染一次，即首次渲染。用户访问过程中，也会不断重新渲染。

  • 修改DOM
  • 修改样式表
  • 用户事件（比如鼠标悬停、页面滚动、输入框键入文字、改变窗口大小等等）

重排：重新生成布局
重绘：重新绘制
重绘不一定会触发重排，但重排一定会触发重绘

提高网页性能就必须减少“重绘”和“重排”，尽量少触发重新渲染。

  • 样式表越简单，重排和重绘就越快。
  • 重排和重绘的DOM元素层级越高，成本就越高。
  • table元素的重排和重绘成本，要高于div元素

提高网页性能的方法

  1. DOM 的多个读操作（或多个写操作），应该放在一起。不要两个读操作之间，加入一个写操作。
  2. 如果某个样式是需要重排生成了，可以考虑进行缓存结果，避免下一次需要重新重排
  3. 不要一条条地改变样式，而要通过改变class，一次性地改变样式。
  4. 先将元素设为display: none（需要1次重排和重绘），然后对这个节点进行100次操作，最后再恢复显示（需要1次重排和重绘）。这样一来，你就用两次重新渲染，取代了可能高达100次的重新渲染。
  5. position属性为absolute或fixed的元素，重排的开销会比较小，因为不用考虑它对其他元素的影响。
  6. 不可见的元素不影响重排和重绘,visibility : hidden的元素只对重绘有影响，不影响重排。

前端性能优化

网络加载

  1. 减少HTTP请求数
    • 从设计实现层面简化页面
    • 资源合并与压缩
    • 合并css图片，使用精灵图
  2. 将CSS和JS放到外部文件中，避免使用style和script标签引入
  3. 异步的加载JavaScript资源
  4. 减少页面重定向
  5. 按需加载

页面渲染

  1. 减少DOM元素数量和深度
  2. 避免运行耗时的JavaScript
  3. 把CSS资源引用放到HTML文件顶部,JavaScript文件引用放到HTML文件底部
